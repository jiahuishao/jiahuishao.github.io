    # 构造函数&原型对象

    构造函数特点：

    • 构造函数的首字母必须大写，用来区分于普通函数
    • 内部使用的this对象，来指向即将要生成的实例对象
    • 使用New来生成实例对象
    function Student(name,sex) {
    
    this.name = name;
    this.sex = sex;
    
    this.sayHello = function() {
      alert("hello" + this.name);
    }
    
    this.girlOrBoy = function() {
      alert("I am a " + this.sex);
    }
}

var red = new Student("red",true);
var blue = new Student("blue",false);

    同一个对象实例之间，无法共享属性

    因此我们引入prototype，将公共部分加入到公有空间中

    Student.prototype.age = 18;

Student.prototype.big = function() {
    this.age++
}

red.age = 18 //true
red.big();
red.age = 19 //true

    • 全局环境的this

    判断 this 指向谁，看执行时而非定义时，只要函数(function)没有绑定在对象上调用， 它的 this 就是 window

    var x = 10;
function func() {
    alert(this.x)
}
var obj = {
    x: 20,
    fn: function() {
        alert(this.x)
    }
};
var fn = obj.fn;
func() ;// 10
fn() ;// 10

    • this 可以被 call/apply 改变

    配合写类工具实现OOP，修复DOM事件里的 this

    var m1 = {
    message: 'This is A'
} 
var m2 = {
    message: 'This is B'
}  
 
function showMsg() {
    alert(this.message)
}
 
showMsg() // undefined
showMsg.call(m1) // 'This is A'
showMsg.call(m2) // 'This is B'

    • ES5 中新增的 bind 和 this

    因为是 ES5 才加的，低版本的IE不支持，可以修复下Function.prototype。bind 只是 call/apply 的高级版，其它没什么特殊的。

    var modal = {
    message: 'This is A'
}
 
function showMsg() {
    alert(this.message)
}
 
var otherShowMsg = showMsg.bind(modal)
otherShowMsg() // 'This is A'

    # computed和watch的区别

    computed主要用于对同步数据的简单处理，watch则主要用于数据输入时异步操作或者开销较大的情况

    computed在监听数据变化时候，上来就会执行一次。而watch页面上来是不会执行的，只有指定数据发生变化才会触发

    computed具有缓存性，值不变化会读取缓存，watch无缓存性，值不变化也会执行
