# 边际效益

边际效益是指增加一单位产品的销售所带来的效益，即最后一单位产品售出所带来的收益。

当边际效益等于0时，利润达到最大值。即利润最大化。

# SWOT分析

SWOT分析拥有两个坐标轴，分别代表，机遇对应风险，优势对应弱势， 两个坐标轴可以明确地划分出四个象限，分别为，SO，ST，WO，WT分别对应优势与机遇， 优势与风险，弱势与机遇，弱势与风险。

SO区战略，企业需要抓住机会以及机遇，利用SO区战略，以谋求更大的发展。 ST区战略，该区中的战略帮助企业降低脆弱项目的风险率。 WO区战略，该区中的战略可以帮助企业抓住机遇，转化劣势。 WT区战略，该区中的战略需要引起企业的重点关注，很可能关系到企业的生死存亡

当进行SWOT分析后，我们应利用SO区的优势，消除WT区的劣势。

总结来说就是三点：
1.第一步：按 OT、SW 的顺序列出 优势、劣势。OT强调外部竞争，SW 强调自身优势劣势
2.第二步：进行组合：按 SO(利用优势、抓住 机会)、WT（扭转劣势，规避风险）、
WO（消除不足抓住机会）、ST（利用优势规避风险），尽可能多的拟出策略；
3.第三步：确定优先级，拟定行动计划

# PEST分析

PEST 为一种企业所处宏观环境分析模型

PEST分别包括政治（Politics），E经济（Economy），S社会（Society），T技术（Technology） 四个环境。

政治环境：

  • 政治环境是否稳定？
  • 国家政策是否会改变法律从而增强对企业的监管并收取更多的赋税？
  • 政府所持的市场道德标准是什么？
  • 政府的经济政策是什么？
  • 政府是否关注文化与宗教？
  • 政府是否与其他组织签订过贸易协定？

经济环境

  • 社会经济结构
  • 经济发展水平
  • 经济体制
  • 宏观经济政策
  • 当前经济状况
  • 其他一般经济条件

社会环境

  • 人口因素
  • 社会流动性
  • 消费心里
  • 消费方式变化
  • 文化传统
  • 价值观

技术环境

  • 科技是否降低了成本提高了利润
  • 科技是否为消费者和企业提供了更多的创新产品与服务，例如网上银行、新一代手机等
  • 科技是如何改变分销渠道的
  • 科技是否为企业提供了一种全新的与消费者进行沟通的渠道

# 波特五力分析

五力分别是： 供应商的议价能力、购买者的议价能力、潜在竞争者进入的能力、 替代品的替代能力、行业内竞争者现在的竞争能力

供应商的议价，供方能力主要通过其提高投入要素价格与降低单位价值质量的能力， 来影响行业中现有企业的盈利能力与产品竞争力

购买者的议价，购方能力主要通过其压价与要求提供较高的产品或服务质量的能力， 来影响行业中现有企业的盈利能力

新进入者的威胁，新入者主要通过瓜分市场份额与原材料市场，来影响行业中的企业 盈利能力

替代品的威胁，替代品通过影响市场份额总量，淘汰旧产品，来影响行业中的企业盈利 能力

行业内竞争者，现有企业之间的竞争常常表现在价格、广告、产品介绍、售后服务等 方面，其竞争强度与许多因素有关

但是波特五力也存在一定的局限性，其的前提是
1.能够清晰的界定行业边界，行业的规模是固定的
2.同行业之间只有竞争关系，没有合作关系
3.能搜集到一切有效信息（信息透明），制定战略者需要了解整个行业的信息

# 波士顿矩阵*

波士顿矩阵存在两个坐标轴，分别为市场占有率以及销售增长率，以此形成四象限图

本企业某种产品绝对市场占有率=该产品本企业销售量/该产品市场销售总量

本企业某种产品相对市场占有率=该产品本企业市场占有率/该产品市场占有份额最大者（或特定的竞争对手）的市场占有率

四象限图坐标轴以10%的销售增长率和20%的市场占有率为高低标准分界线

  • 销售增长率和市场占有率“双高”的产品群（明星类产品）
  • 销售增长率和市场占有率“双低”的产品群（瘦狗类产品）
  • 销售增长率高、市场占有率低的产品群（问题类产品）
  • 销售增长率低、市场占有率高的产品群（金牛类产品）

明星类产品：此类产品拥有高销售增长率以及高市场占有率，拥有非常好的前景，在日后 极有可能成为金牛类产品。对待这种产品企业应该积极投资，以长远利益为目标，提高市场 占有率，巩固自身地位。发展战略以及明星产品的管理与组织最好采用事业部形式， 由对生产技术和销售两方面都很内行的经营者负责。

瘦狗类产品：此类产品也被称为衰退类产品。处在低市场占有率以及低增长率，其特点是 产品通常位于保本或者亏损状态下，对待这种产品，我们应采取撤退战略，逐步退出市场 对于双极低产品应立即淘汰。将其所占有资源向其他项目转移，整顿其事业部，与其他事业部 合并。

现金牛类产品：此类产品，处于高市场占有率，但是其销售额增涨却非常缓慢甚至停滞， 此类产品的作用在于为其他产品业务提供资金支持，因此企业对这种产品业务的设备投资 应进行压缩。目标是维持其当前所拥有的市场占有率。对于现金牛产品，适合于用事业部制进行管理， 其经营者最好是市场营销型人物。仅仅拥有一个现金牛产品的公司是非常脆弱的，市场上 的任何一个变动，都可能导致其迅速衰退。使公司不得不从其他业务中抽取资金来维持他的 其的领导地位，否则其非常容易转变成瘦狗类产品。

问题产品：它是处于高增长率、低市场占有率象限内的产品群。此类产品的前景好，但是同时 其利润较低，甚至负债。此类产品应该有选择性的投资。应该例如的企业的长期计划中。对问题产品的管理组织， 最好是采取智囊团或项目组织等形式，选拔有规划能力，敢于冒风险、有才干的人负责。

运用：

  • 发展：以提高经营单位的相对市场占有率为目标．甚至不惜放弃短期收益。要使问题类业务想尽快成为“明星”，就要增加资金投入。
  • 保持：投资维持现状，目标是保持业务单位现有的市场份额、对于较大的“金牛”可以此为目标，以使它们产生更多的收益。
  • 收割：这种战略主要是为了获得短期收益，目标是在短期内尽可能地得到最大限度的现金收入。对处境不佳的金牛类业务及没有发展前途的问题类业务和瘦狗类业务应视具体情况采取这种策略。
  • 放弃：目标在于清理和撤销某些业务，减轻负担，以便将有限的资源用于效益较高的业务。这种目标适用于无利可图的瘦狗类和问题类业务。一个公司必须对其业务加以调整，以使其投资组合趋于合理。

# 市场规模测算-源推算法

目标行业市场规模 = 源行业市场规模 / 源行业产品均价 * 目标行业产品均价 * 两行业产品数配比

# 市场规模测算-强相关数据推算法

目标行业市场规模 = 相关行业市场规模 / 相关行业产品均价 * 目标行业产品均价 * 两行业产品数配比

# 市场规模测算-需求推算法

目标市场规模 = 目标需求人群数量 * 购买率 * 目标行业产品均价

# 市场规模测算-抽样分析法

目标市场规模 = 购买数 / 样本数 * 购买个数 * 样本平均价格 * 总体数

# 市场规模测算-典型反推法

目标市场规模 = 单品牌销售额 / 该品牌市场份额

# 马斯洛需求理论

马斯洛需求理论将人的需求分成五类，从低到高分别为，初级需求（生理需求，安全需求），中级需求（社会需求，尊重需求），高级需求（自我实现需求） 但是，这并不是固定的，不同场景下其等级会发生变化。

生理需求，分别包括：呼吸，水，食物，睡眠，生理平衡，分泌，性，如果 某一样打不到需求（出去性），那么人的生命就会收到威胁在这个意义上说，生理需要是推动人们行动最首要的动力。马斯洛认为，只有这些最基本的需要满足到维持生存所必需的程度后，其他的需要才能成为新的激励因素，而到了此时，这些已相对满足的需要也就不再成为激励因素了。

安全需求，包括：人生安全，健康保障，资源所有性，财产所有性，道德保障，工作职位保障，家庭安全 马斯洛认为，整个有机体是一个追求安全的机制，人的感受器官、效应器官、智能和其他能量主要是寻求安全的工具，甚至可以把科学和人生观都看成是满足安全需要的一部分。当然，当这种需要一旦相对满足后，也就不再成为激励因素了。

社会需求，包括：友情，爱情，性亲密。人人都希望得到相互的关系和照顾。感情上的需要比生理上的需要来的细致，它和一个人的生理特性、经历、教育、宗教信仰都有关系。

尊重需求，包括：自我尊重，信心，成就，对他人尊重，被他人尊重。人人都希望自己有稳定的社会地位，要求个人的能力和成就得到社会 的承认。尊重的需要又可分为内部尊重和外部尊重。内部尊重是指一个人希望在各种不同情境中有实力、能胜任、充满信心、能独立自主。总之，内部尊重就是人的自尊。外部尊重是指一个人希望有地位、有威信，受到别人的尊重、信赖和高度评价。马斯洛认为，尊重需要得到满足，能使人对自己充满信心，对社会满腔热情，体验到自己活着的用处价值。

自我实现需求，道德，创造力，自觉性，问题解决能力，公正度，接受现实能力。自我实现的需要是最高层次的需要，是指实现个人理想、抱负，发挥个人的能力 到最大程度，达到自我实现境界的人，接受自己也接受他人，解决问题能力增强，自觉性提高，善于独立处事，要求不受打扰地独处，完成与自己的能力相称的一切事情的需要。也就是说，人必须干称职的工作，这样才会使他们感到最大的快乐。马斯洛提出，为满足自我实现需要所采取的途径是因人而异的。自我实现的需要是在努力实现自己的潜力，使自己越来越成为自己所期望的人物。

理论总结与即系
1.五种需求，从高到低，逐级上升，但是次序并不固定
2.只有当前一种需求得到满足，后一种需求才会出现
3.初级需求通过外部满足，中级需求通过内部满足，自我实现需求永无止境
4.一个国家的需求层级结构和发展水平有关，越发达拥有越高级需求的人越多

在公司企业中每种需求的激励措施

  • 生理需求：增加工资、改善劳动条件、给予更多的业余时间和工间休息、提高福利待遇。
  • 安全需求：强调规章制度、职业保障、福利待遇，并保护员工不致失业，提供医疗保险、失业保险和退休福利、避免员工收到双重的指令而混乱。
  • 社交需求：提供同事间社交往来机会，支持与赞许员工寻找及建立和谐温馨的人际关系，开展有组织的体育比赛和集体聚会。
  • 尊重需求：公开奖励和表扬，强调工作任务的艰巨性以及成功所需要的高超技巧，颁发荣誉奖章、在公司刊物发表文章表扬、优秀员工光荣榜。
  • 自我实现需求：设计工作时运用复杂情况的适应策略，给有特长的人委派特别任务，在设计工作和执行计划时为下级留有余地。

消费市场的划分

  • 生理需求：满足最低需求层次的市场，消费者只要求产品具有一般功能即可
  • 安全需求：满足对“安全”有要求的市场，消费者关注产品对身体的影响
  • 社会需求：满足对“交际”有要求的市场，消费者关注产品是否有助提高自己的交际形象
  • 尊重需求：满足对产品有与众不同要求的市场，消费者关注产品的象征意义
  • 自我实现需求：满足对产品有自己判断标准的市场，消费者拥有自己固定的品牌 需求层次越高，消费者就越不容易被满足。

经济学上，“消费者愿意支付的价格≌消费者获得的满意度”，也就是说，同样的洗衣粉，满足消费者需求层次越高，消费者能接受的产品定价也越高。市场的竞争，总是越低端越激烈，价格竞争显然是将“需求层次”降到最低，消费者感觉不到其他层次的“满意”，愿意支付的价格当然也低。

# Kano模型

kano模型被大量用于用户调查分析问卷

kano模型拥有两个维度，分别为用户满意度，以及产品具备程度。

kano模型一共将需求分为五类，分别为，基本型需求，期望型需求，魅力型需求，无差异需求，逆向需求

  • 基本型需求：用户不会因为基本型需求被满足而无限提高自己的满意度，知识满意度无限 趋近于临界值，即0

  • 期望型需求：需求型需求被满足越多，用户的满意度越高，基本成等比增涨

  • 魅力型需求：用户不会因为没有魅力型需求而无限降低自己的满意度，即无限趋近于临界值 ，即0

  • 无差异需求：此种需求不会影响用户的满意度

  • 逆向需求：逆向需求被满足的越多，用户的满意度越低，

    1.需求会因人而异，大多数情况下我们只满足大多数人的需求 2.需求会因为文化差异而不同 3.需求会随着时间的变化而变化，现在的魅力型需求可能在未来就会变为基本型需求

kano模型的调查问卷中，将满意度分为5个等级，一个基本的满意度问卷如下所示

问题 满意度高 满意度较高 满意度0 满意度较低 满意度低
如果产品/服务有**模块，您的评价是 - - - - -
如果产品/服务没有**模块，您的评价是 - - - - -

通过kano模型，可以甄别需求阶段，对伪需求进行剔除

# 用户需求法则

  • 需求是用户对解决现存问题的需要

    • 我们寻找的需求是线下能被用户买账的需求，不是50年，100年以后的需求 但是同时，这个需求也需要有发展性，不能是一次性需求

  • 需求不是无边界的，满足超过一定边界，边际效益会骤降

    • 任何需求都有边界的，如果无止境得去满足一些需求，其所带来的利润会骤 降。例如用户需要更快的交通工具，你为了满足这个要求给他们加上了火箭。 那么你的收益就会骤降。

  • 用户的诉求并不等于用户的需求

    • 用户往往不能非常明确地说出自己的需求。例如用户要个汽车，但是他的目的 可能并不是买个汽车，可能是为出行提速，可能是满足家庭出行需要，或者是炫富 或者等等其他的。所以我们要从大量的用户诉求中，分析出用户真正的需求是什么

  • 需求的主体是目标用户

    • 在做内部需求收集时，常常会接触到员工，经理，最后到老板，但是他们的需求 可能并不是你的目标用户的需求。所以要明确你的目标人群到底是是什么。

  • 需求有其时空约束

    • 每个需求都有它的作用场景，不同场景用户的需求很可能产生一定的变化，所以 要明确，你的产品的使用环境。

  • 用户是需求的集合

    • 关注用户时，要知道，一个用户往往拥有多个需求，是一个需求的集合体，就好比 微信和手电筒，假设他们的用户数量都是一千万，但是微信作为一个反馈需求的集合 体其价值，远远超过可能只是偶尔使用的手电筒。所以不能简单地用用户数量来判定 一个产品。

  • 需求存在不同层次

    • 有的需求往往存在着一个更深的需求，优点类似于用户的诉求，但是也不尽相同 深层的需求往往可以连带着满足多个需求。

# M-P能力模型

productManage，产品经理的能力主要由两部分组成，第一部分是市场运作能力，第二部分 是需求生成能力。产品经理的等级越高，对其的市场运作能力要求越高，对产品战略决策层面 辨识能力要求越高。

  • 需求生成能力-需求翻译

    • 主要是将一个非常抽象，涵盖面非常广的需求，逐步拆解逐步拓展，最后形成一个能够 落地的app

  • 需求生成能力-项目管理

    • 总结就是看一个产品经理是否能做到让app在deadline拥有完备的功能，同时上线

  • 市场运作能力-市场推广

    • 如何将产品推向市场接触目标用户，并占领市场

  • 市场运作能力-商业变现

    • 通过一定的手段，一定的渠道，制定收费模式，达到变现目的

产品经理的几个类型

  • 战略型
  • 筹划型
  • 高级执行
  • 低级执行

谈论每个类型的产品经理对应的职责前，我们将一个产品按照如下方式进行从大到小 的拆分

  • 战略层面：这个产品的商业变现以及市场推广的维度，也就是如何进行推广，我们需要投入 多少资金，期望可以收获多少利润。简而言之就是思考目标和关键时间点
  • 筹划层面：这个产品如何吸引用户，吸引什么类型的用户，针对这些用户我们需要退出什么 样的功能，具体点就是，产品的主次功能，当然这个层面所有的决策都基于战略层面的规划。
  • 梳理大致流程：就是更具筹划层面划分的主次功能，开始屡清楚每个功能的流程，在每个 功能中是否需要填加子功能，每个功能之间又有什么样的关系。
  • 拆解层面：具体到每个流程的页面原型，用户交互，每个功能需要如何去实现，如何安排 进度。

这四个层面就分别对应上文提到的几个不同类型的产品经理。

其实归根结底就是负责的对象不同，低等级对产品负责，高等级对股东负责。

# 需求定义

# 经济学定义

人们有能力购买并且愿意购买某个具体商品的欲望被称为需求

# 需求拆解

期望 ————> 具象行为 ————> 核心需求

# 需求分类

需求从程度上分，可以分为，痛点，痒点，兴奋点，从真实性上分，可以分为，真需求，伪需求

# 构成需求的要素

需求 = 用户 + 场景 + 诉求 + 任务

用户画像：用户主要可以分为三类。第一类，对自己的需求明确，价格敏感，依赖搜索， 没有忠诚度。第二类，对自身需求不够清晰，但是明确需求后立刻转变成为第一类。第三类， 没有明确需求，打发时间。

# 死亡三连

到底解决什么需求？ 问题严不严重？ 多少用户会遇到？

# 需求来源

行业/用户，竞争对手，老板，相关业务部门，头脑风暴

# 头脑风暴四个原则：
  • 不得对任何成员的发言作评判
  • 打破思维限制，天马行空去思考
  • 不管好坏，电子越多越好
  • 借鉴别人的观点，无需考略
# 头脑风暴方向：
  • 否定
  • 积极
  • 转移
  • 脑洞大开
  • 分解

# 需求到实现

诉求（需求来源） ————> 需求（生成确认需求） ————> 产品（需求评审） ————> 功能（需求落地）

产品和竞品对基本型需贡献较大，需要产品经理们在实践中提升对用户以及行业的洞察力形成核心竞争

# 用户需求采集的方式

  • 用户访谈

# 设计访谈的问题

访谈的问题一共分为以下几类：事实性问题（描述正在发生的客观问题） ，行为性问题（具体完成的情况），倾向性问题（不同选择的偏好） ，原因性问题（背后的原因），主观性问题（对事物的看法），建议性问题，每种问题 控制在10个以内，访谈时间控制在半个小时到一个小时之间。

访谈时需要注意倾听，重复对方的话进行核实，同时认可和鼓励对方。不要使用带有非常 明显的引导性的词，不要随意评论用户的价值。

# 问卷调差

问卷调差可以进行两到三轮，此次递进，数量也逐级增加。简单的问题排在前面，复制 的问题放在后面，开放性的问题放在后面。问卷需要有可理解性，逻辑性，真实性，避免 双重含义。

问卷的问题模块可以分为三个，分别为：客观背景，行为记录，需求挖掘。客观背景抱愧用户 的性别，年龄层，学历背景，平时休闲行为，用户的工作，工作类型，工作时间。行为记录包括 操作行为，操作频次，态度反馈，消费情况，以及对应的竞品调查。需求挖掘主要寻找用户的诉求 ，从中整合出需求。

# 需求管理

# 需求池

需求池是方便我们和各方沟通的标准化，规范化的工具

需求池包含字段：序号，所属模块，需求来源（运营，用户访谈，业务发展）， 需求类别（迭代类，异常类，项目类），需求类型（用户需求，运营需求，业务需求，竞争需求）， 处理状态（未开始，已评估，已排期，开发中，已上线）， 优先级（p0,p1,p2,p3），预计完成时间，实际完成时间，排入版本，接口人，备注。

1.需求类别：区分条件如下，迭代类工作量在5-50人日左右，异常类工作量在
5人日以内，项目类工作量在50人日以上。
2.优先级的选定需要参考如下几个关键因素，需求价值，需求工作量，需求紧急度

# 伪需求

伪需求一共有几个以下的特征，1.不存在 2.不合理 3.非本质

1.不存在，初级的产品经理没有经验，总是做出拍脑袋的需求，这类需求往往不依赖
任何数据或者调研报告用户反馈，因此容易不存在。
2.不合理，这类需求往往因为各个因素问题，无法得到解决，这些因素可能是技术
原因，也可能是成本原因或者其他等等
3.非本质，这类需求往往没有触及到问题的核心，需要产品对问题进行深层次的考虑

# 产品生命周期

分为初始期（策略：可用，特色），成长期（策略：易用，爱用），成熟期（策略：好用，传播）， 衰退期（策略：拓展，孵化）。

1.初始期需要的是吸引用户，很多时间，用户对你搬出的凳子都是视而不见的，
这时候不管你产品做得再好，再优秀也无法请用户进门，因此你需要将你的产品
做出特色，用特色来撬开用户的好奇心，同时要做好基本功能的保障，来巩固
留存用户的量
2.成长期，你需要带给用户更好地体验，如果基本功能知识留存量的们，只有做好
易用性，才能增加永久中间用户的占比
3.成熟期，面对永久中间用户的占比不断扩大，你需要满足他们日渐刁钻的口味，
同时因为新手用户的不断转化，其基数还是减小，你需要考虑为你的铲平带来
一定的新鲜血液，因此好用，传播是这个阶段的主旋律。
4.衰退期，这个时期你已经组织不了用户的流失，这样你要考虑的就是如何为
该产品留下资源，来服务孵化下一个产品。

# 需求池管理常见思路

严进快出（适合快速开发），宽进严出（通用型，适合各个阶段），矩阵式（整合相同类似需求）

# 游戏化设计

# 正向驱动力

# 1.责任与使命

让用户认为，或者潜意识中认为他的一举一动都能够产品很大的影响，远远比他们自己认为 的重要，例如，wiki百科

# 左脑驱动力

# 2.进步与成就感

让用户体验到类似完成任务，等级提升的成就感，引导用户去增加使用产品的时间。 案例，网易云音乐的等级系统，qq等级以及勋章系统。

# 3.所有权和占有欲

告诉用户他这样做可以拥有更多地东西，以此彰显用户的独特性和阶级性。案例， 农场，蚂蚁森林。

# 4.稀缺性与渴望

过了这村，就没这店，数量有限，欲购速从。和游戏中的稀有装备一样，用稀缺性引导 用户进行某种操作。案例，lol限时皮肤，各种商城打折。

# 右脑驱动力

# 5.创造力与反馈

引导用户去开发属于自己的创意，同时让其看到他的操作被大众所接受，浏览，能让其 产生一种第一无二性。案例，抖音。

# 6.关系与连结性

作为一个社会性动物，与他人有所交集，能够留存非常大一部分用户，就和著名社交游戏 剑网三一样，垃圾游戏，要不是因为很多朋友都在里面，我早不玩了。案例，大多数社交 软件，soul，陌陌等等

# 7.未知与好奇心

永远对用户保持一定的神秘性，让他们觉得产品还有没有被发现的功能，用户或者消息， 案例，微信朋友圈，微博，知乎等等。

# 负向驱动力

# 8.损失与逃避心里

利用人类，害怕失去，原理压力的心里。案例，百度云盘的加速，还有小米的饥饿营销

# 游戏化设计的应用

用户的产品使用成长周期可以大致分为，发现，新手入住，深入学习，习惯成瘾 在发现阶段的核心驱动力主要是未知与好奇心，稀缺性与渴望，责任与使命感， 新人入住阶段影响较大的因素是进步与成就感，关系与连结性，责任与使命感， 第三阶段也就是学习深入阶段主要为进步与成就感，损失与逃避心里，创造力与反馈， 习惯成瘾阶段左右用户的主要因素有所有权与占有欲，损失与逃避心里，责任与使命感。
